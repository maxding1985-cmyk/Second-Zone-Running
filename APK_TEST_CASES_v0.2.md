# 跑步聊天 Android APK v0.2 语音说话测试用例

> 版本定位: 语音优先的二区说话测试 MVP  
> 测试目标: 验证用户必须开口说话，App 能转写、分类、语音反馈，并生成语音报告  
> 注意: v0.2 的核心验收不是按钮能不能点，而是“跑中说话测试”是否跑通。

---

## 1. 核心正确输出速查

| 场景 | 输入 | 正确输出 |
| --- | --- | --- |
| 跑前教育 | 打开开始流程 | 看到“跑到还能说完整句”“能说完整句子，但不能舒服唱歌” |
| 麦克风权限 | 首次进入说话测试 | 系统请求麦克风/语音识别权限 |
| 正常说话 | “我现在感觉还可以，呼吸有点快，但是能完整说话。” | 分类 `target`，AI 提示保持当前节奏 |
| 有点喘 | “我有点喘，说话要停一下。” | 分类 `breathless`，AI 提示慢一点、缩短步幅 |
| 太累 | “我说不完整，太累了，喘不过来。” | 分类 `too_hard`，AI 提示先走路，等呼吸恢复 |
| 风险词 | “我有点胸痛和头晕。” | 分类 `risk`，AI 提示马上停止运动并寻求帮助 |
| 语音失败 | 不说话/识别失败 | 显示没有听清或准备重试，不计为有效说话测试 |
| 报告 | 完成跑步 | 展示语音回应/说话测试、完整句次数、STT 失败、按钮 fallback |
| 跑后问卷 | 完成报告页问卷 | 可选择二区理解、语音有用性、复用意愿、真人陪跑兴趣 |

## 2. P0 用例

### TC-V02-01 跑前教育

| 字段 | 内容 |
| --- | --- |
| 前置条件 | App 已安装 |
| 操作 | 首页点击“开始聊天跑” |
| 正确输出 | 进入跑前页，看到 v0.2 说话测试教育 |
| 必须出现文案 | “跑到还能说完整句”“能说完整句子，但不能舒服唱歌”“如果只能挤出几个词、说话要停下来喘，就说明该慢一点” |
| 通过标准 | 用户不需要外部解释，就能理解本次不是点按钮，而是开口说话 |

### TC-V02-02 正常说话测试

| 字段 | 内容 |
| --- | --- |
| 前置条件 | 已进入跑中页，已允许麦克风权限 |
| 操作 | 点击“现在开始说话测试” |
| 测试输入 | 对手机说：“我现在感觉还可以，呼吸有点快，但是能完整说话。” |
| 正确输出 | 页面显示转写文本；判断为 `target`；AI 语音提示保持节奏 |
| 通过标准 | 不点击体感按钮也能完成本轮核心互动 |

### TC-V02-03 有点喘降速提醒

| 字段 | 内容 |
| --- | --- |
| 前置条件 | 已进入跑中页 |
| 操作 | 开始一次说话测试 |
| 测试输入 | “我有点喘，说话要停一下。” |
| 正确输出 | 分类 `breathless`；AI 提示“慢一点/缩短步幅/回到能完整说话” |
| 通过标准 | 报告里的语音降速提醒次数 +1 |

### TC-V02-04 安全风险兜底

| 字段 | 内容 |
| --- | --- |
| 前置条件 | 已进入跑中页 |
| 操作 | 开始一次说话测试 |
| 测试输入 | “我有点胸痛和头晕。” |
| 正确输出 | 分类 `risk`；AI 提示马上停止运动，必要时寻求帮助 |
| 通过标准 | 报告里的安全兜底次数 +1；不继续普通鼓励话术 |

### TC-V02-05 不点按钮完成语音报告

| 字段 | 内容 |
| --- | --- |
| 前置条件 | 已完成至少 1 次说话测试 |
| 操作 | 点击“结束并生成语音报告” |
| 正确输出 | 报告页显示“这次核心是：你有没有开口说话。” |
| 正确统计 | 语音回应/说话测试、完整句次数、STT 失败、按钮 fallback |
| 通过标准 | 按钮 fallback 为 0 时，仍然能生成报告 |

### TC-V02-06 跑后假设验证问卷

| 字段 | 内容 |
| --- | --- |
| 前置条件 | 已进入报告页 |
| 操作 | 填写所有问卷项并提交 |
| 测试输入 | 理解了 / 有帮助 / 愿意 / 轻松 / 可以看看 / 任意评论 |
| 正确输出 | 按钮显示“已提交反馈” |
| 通过标准 | 运营看板可统计二区理解率、语音有用率、真人兴趣率 |

## 3. Fail 条件

出现以下任一情况，v0.2 不通过：

1. 跑中主要交互仍要求用户点击“轻松/有点喘/太累”。
2. 用户没有说话也被算作核心 MVP 有效样本。
3. 用户说“有点喘”后，App 没有提醒慢下来。
4. 用户说“胸痛/头晕/心悸/呼吸困难”后，App 没有安全停止提示。
5. 报告不展示语音回应、完整句、STT 失败、按钮 fallback。
6. 跑后问卷没有收集二区理解和真人陪跑兴趣。
